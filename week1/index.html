Algebraic Data Types

  • not strictly "functional"
  • best use of pattern matching
  • no more NullPointerExceptions!

The Algebra in Algebraic Data Types

  • sums vs products
  • most languages have product types, but not sum types

    • A struct in C is a product type: 

    struct Point { int x; int y; };
    Maybe (Haskell) is a sum type: 
    data Maybe a = Nothing | Just a

Why sums matter

  • Make the bad state unrepresentable
  • eliminate entire classes of errors
  • better refactoring
    • Person p = findPerson(name);
if (p != null) {
    deductTaxes(p);
} else {
    notifyHrNonExistant(name);
}
    vs 
    case findPerson name of 
  Just p  -> deductTaxes p 
  Nothing -> notifyHrNonExistant name
